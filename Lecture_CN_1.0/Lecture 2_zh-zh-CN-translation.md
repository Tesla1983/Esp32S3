## 探索 ESP 微控制器

### 介绍

微控制器是功能强大的设备，让我们能够构建智能化和自动化系统。虽然像 Arduino 这样的平台降低了电子学习的门槛，但现代世界对互联性提出了更高要求。如今的项目往往需要连接互联网、向云端传输数据，或与智能手机通信。为传统基础微控制器添加这些功能过去既复杂又昂贵。

为了解决这一问题，乐鑫科技推出了一系列低成本、高度集成的微控制器，彻底改变了物联网（IoT）领域：即 ESP 系列。

### 什么是 ESP？

ESP 系列由低功耗、高度集成的微芯片组成，内置 Wi-Fi 和蓝牙功能。无需分别使用独立的微控制器和无线模块，ESP 芯片将两者集成于单一硅片中。

凭借其强大的处理能力、充足的存储空间以及内置的无线功能，ESP 已成为物联网设备、智能家居自动化和高级 DIY 电子产品的行业标准。此外，它们还得到了开源社区的大力支持，使得初学者的入门过程异常顺畅。

### 裸片 vs. 模块 vs. 开发板

ESP 产品通常以三种不同形式出现。理解这一层级结构至关重要：

#### 裸片：

这是未经封装的微型硅芯片本身（例如 ESP32-D0WDQ6）。直接使用它非常困难，因为需要围绕它设计复杂的电路，包括外部闪存、晶振和天线。

![](./attachments/bare_chip.png)

#### 模块：

为了让芯片可供制造商使用，乐鑫将裸芯片与闪存、晶振和集成天线（通常由闪亮的金属屏蔽罩覆盖）一同封装在一块邮票大小的电路板上。一个著名的例子是 **ESP-WROOM-32** 模块。虽然更易于使用，但它仍需焊接至定制电路板，且严格工作在 3.3V 电压下。

![](./attachments/esp-wroom-32.png)

#### 开发板（Dev Kit）：

开发板将 ESP 模块焊接在一块更大的、兼容面包板的电路板上。它增加了 USB 端口、USB 转串口转换器、3.3V 稳压器以及易于连接的排针，使其成为学习和原型开发的最佳选择，因为供电、编程和连接其他组件都更加方便。

![](./attachments/DevKit.png)

### ESP 系列

乐鑫科技已发布多代芯片，每一代都针对不同的技术需求进行了定制。

#### ESP8266

ESP8266 是开启物联网革命的芯片。在其问世之前，为微控制器添加 Wi-Fi 功能成本高达 2 美元以上。它搭载单核 32 位处理器并内置 Wi-Fi 模块。虽然这款芯片较为老旧，且输入/输出引脚数量少于后续产品，但在简单、低成本的智能家居传感器和开关领域仍广受欢迎。该芯片的常见开发板包括 NodeMCU 和 Wemos D1 Mini。

![](./attachments/esp8266.png)

#### ESP32

最初的 ESP32 是 8266 的强大继任者。它几乎升级了所有方面：配备双核 32 位处理器，增加了经典蓝牙和低功耗蓝牙，大幅增加了 GPIO 引脚数量，并包含硬件加密。对于需要高负载处理的项目（如音频流、复杂网络服务器或同时管理多个传感器），它是黄金标准。

#### ESP32-S 系列（S2、S3）

ESP32-S 系列专注于安全性、专用功能和高级处理能力。

*   **ESP32-S2：** 一款单核 Wi-Fi 芯片，原生支持 USB 功能，可模拟鼠标或键盘操作，并增强了安全特性。
*   **ESP32-S3：** 专为边缘端人工智能和机器学习设计的双核芯片。其内置的"向量指令"能显著加速神经网络运算，非常适合语音识别或人脸检测摄像头等应用。

![](./attachments/esp32s3.png)

#### ESP32-C 系列

ESP32-C 系列标志着乐鑫设计理念的转变。与早期基于专有 Xtensa 架构的芯片不同，C 系列采用开源 RISC‑V 架构。这些芯片专注于低功耗、高性价比和现代无线连接，是物联网设备的理想选择。

*   **ESP32-C2：** 一款入门级、超低成本的微控制器，专为简单的物联网应用而设计。它提供 Wi-Fi 4（2.4 GHz）连接和基本安全功能，同时保持极低功耗，适用于大规模生产的联网设备。
*   **ESP32-C3：** 一款广受欢迎且广泛使用的芯片，可作为 **ESP8266** 的现代替代品。它采用单核 RISC-V CPU，支持 Wi-Fi 4 和蓝牙 LE 5.0，为通用物联网项目提供更高的安全性、性能和低功耗。
*   **ESP32-C5：** 引入双频 Wi-Fi 6（2.4 GHz 和 5 GHz）支持，相比早期芯片提供更高的吞吐量和更低的延迟。它专为需要更快、更可靠连接的更高要求的无线应用而设计。
*   **ESP32-C6：** 通过结合 Wi-Fi 6、蓝牙 LE 5.3 以及对 Zigbee 和 Thread 协议的支持，瞄准下一代物联网生态系统。这使得它特别适用于需要多种无线标准的智能家居和 Matter 兼容设备。

#### ESP32-H 系列

ESP32-H 系列专为现代智能家居生态系统中使用的低功耗网状网络而设计。这些芯片专注于 IEEE 802.15.4 协议，而非 Wi-Fi。

*   **ESP32-H2：** 一款基于 RISC-V 的低功耗微控制器，专为依赖 Thread、Zigbee 及其他 IEEE 802.15.4 协议的设备打造。它还集成了蓝牙 LE 5.2，用于设备配置和通信。与大多数 ESP32 芯片不同，它不包含 Wi-Fi，因此非常适合兼容 Matter 的智能家居设备、传感器以及电池供电的网状网络节点。
    
*   **ESP32-H4：** 设计为面向物联网系统的蓝牙专用伙伴微控制器。它针对低功耗蓝牙应用，如可穿戴设备、传感器和网关。H4 可与其他 ESP 芯片（例如 Wi-Fi 微控制器）配合使用，以分担蓝牙连接和无线协议处理任务，从而提高多无线电系统的效率。
    

#### ESP32-P 系列

ESP32-P 系列面向需要比典型物联网微控制器更强大处理能力的应用场景。

*   **ESP32-P4：** 一款基于 RISC-V 架构的强大 MCU，专为计算密集型嵌入式系统设计，例如高级人机界面、多媒体处理以及边缘 AI 工作负载。与大多数 ESP 芯片不同，它不包含内置无线连接功能，从而能够完全专注于 CPU 性能、内存带宽和外设能力。无线连接可通过使用配套芯片来添加。

### ESP32-S3 结构

ESP32-S3 是一款专为 AIoT（人工智能物联网）应用设计的高性能微控制器。它在单芯片上集成了无线连接、AI 加速功能以及丰富的外设资源。

了解其内部结构有助于我们理解它如何处理数据、与设备通信以及连接网络。

要理解 ESP32-S3 的工作原理，我们需要拆解其核心组件：

![](./attachments/esp32s3pins.png)

#### 微控制器核心

ESP32-S3 搭载双核 Xtensa LX7 处理器，运行速度最高可达 240 MHz。
该处理器作为系统的大脑，负责执行程序指令、处理传感器数据、执行计算任务，并控制显示屏、电机及通信接口等输出设备。

由于采用双核架构，任务可被高效分配。例如，一个核心可管理通信或后台进程，而另一个核心则处理应用逻辑或信号处理。

#### 内存结构

ESP32-S3 包含 512 KB 的内部 SRAM，在程序运行时用作工作内存。

除了内部存储器外，该芯片还支持外部高速 SPI 闪存和 PSRAM。这些外部存储器使开发者能够存储更大规模的程序、缓冲区以及更复杂应用（如图像处理或 AI 工作负载）所需的数据集。大多数 ESP32 模块配备 4 MB 至 16 MB 的大容量闪存。

#### Wi-Fi 与蓝牙连接功能

ESP32-S3 的关键特性之一是其集成的无线通信能力。

它内置了 2.4 GHz Wi-Fi（802.11 b/g/n）用于网络连接，以及蓝牙 5 低功耗（LE）技术，用于设备间的短距离无线通信。

蓝牙低功耗（LE）支持现代功能，例如使用编码物理层（coded PHY）实现远距离通信、通过 2 Mbps 物理层（2 Mbps PHY）实现更高数据吞吐量，以及扩展广播能力。这些无线技术使 ESP32-S3 能够轻松连接智能手机、传感器、云服务及其他智能设备。

#### AI 加速

ESP32-S3 在处理器内部设计了向量指令支持。这些指令可加速信号处理和神经网络计算中常用的运算操作。

这种硬件加速能力使开发者能够直接在设备上运行轻量级机器学习模型。ESP-DSP 和 ESP-NN 等库提供了优化后的实现方案，让构建语音识别、手势检测和计算机视觉等 AI 应用变得更加便捷。

#### 输入/输出外设

ESP32-S3 配备了丰富的外设接口，能够与各类电子组件进行交互。

它提供多达 45 个可编程 GPIO 引脚，这些引脚可配置为不同功能，能够连接传感器、显示屏、存储设备及通信模块。

该芯片还支持多种通信接口，包括：

*   **SPI**
*   **I2C**
*   **I2S**
*   **UART**
*   **PWM**
*   **RMT**
*   **SD/MMC 主机**

此外，14 个 GPIO 引脚可用作电容式触摸输入，从而能够构建基于触摸的人机交互界面。

#### 模拟功能

ESP32-S3 内置模数转换器（ADC），使系统能够读取来自温度传感器、光敏传感器或电位计等设备的模拟电压信号。

该 ADC 将这些模拟电压转换为微控制器可通过软件处理的数字值。

#### 超低功耗（ULP）协处理器

为支持节能应用，ESP32-S3 内置了一个超低功耗协处理器。

该小型处理器可在主核心处于休眠模式时运行。它能够以极低的功耗监控传感器、采集数据或触发唤醒事件。此特性对于电池供电设备及长期监测系统尤为实用。

### ESP-IDF

要为 ESP32-S3 及其他 ESP 芯片开发应用程序，我们使用 **ESP-IDF**（乐鑫物联网开发框架）。
ESP-IDF 是乐鑫系统为 ESP32 系列微控制器编程提供的官方开发框架。它是一个专为专业及高级嵌入式系统开发设计的强大环境。

ESP-IDF 提供了编写、编译和上传固件至 ESP 设备所需的工具。它包含一系列库、驱动程序和开发工具，使开发者能够访问微控制器的硬件功能，如 GPIO、Wi-Fi、蓝牙、定时器和通信接口。

该框架可从官方网站及文档下载：
[https://docs.espressif.com/projects/esp-idf/en/latest/esp32/](https://docs.espressif.com/projects/esp-idf/en/latest/esp32/)

ESP-IDF 还包含许多示例项目，帮助用户了解如何使用 ESP 芯片的不同外设和功能。这些示例演示了诸如连接 Wi-Fi 网络、读取传感器数据、控制显示屏以及使用蓝牙通信等任务。此外，该框架还提供了内置的组件库，简化了硬件模块的操作，使开发者能够更高效地构建复杂的物联网应用。

## ESP32 编程

### 介绍

要使用 ESP-IDF 框架对 ESP32 微控制器进行编程，我们主要使用 C 和 C++语言。项目结构通常包含一个 `ProjectName.c` 文件，作为应用程序的入口点。它还包含构建配置文件，这些文件将源代码编译为微控制器可执行的二进制指令。

`ProjectName.c` 文件包含 `app_main()` 函数，该函数在 **ESP32** 开发板通电或复位时运行一次。在此函数内部，我们放置希望微控制器执行的指令。如果需要一组指令持续运行，可以加入 `while(1)` 循环，使代码不断重复执行。

在开始之前，让我们先探索 C 语言的关键结构和语法。

### 变量与数据类型

C 语言中的变量可以理解为用于存储程序可操作信息的容器或盒子。每个变量都有数据类型，用于告知编译器它将存储何种数据，例如数字、文本或逻辑状态。

要创建一个变量，我们首先声明变量的数据类型，然后是变量名，如果它有值，则使用 `=` 运算符为其赋值：

```c
类型 名称 = 值;
```

ESP-IDF 支持标准 C 数据类型：

#### `char`

表示单个字符，例如 `'a'`、`'!'` 或 `'$'`。

*   占用 1 字节内存。
*   使用 ASCII 编码存储。
*   对 `char` 进行算术运算时，实际是对其 ASCII 值进行操作。

```c
char letter = 'A';  // ASCII 值 65
```

#### `char` 数组

表示一个字符序列（文本），例如 "Hello" 或 "ESP32"。在 C 语言中，字符串实际上是以空字符（`\0`）结尾的字符数组。示例：

```c
char message[] = "Hello ESP32";
```

#### `int`

表示整数，例如 `5`、`-10` 或 `0`。

*   由于 ESP32 是一款 32 位微控制器，`int` 类型占用 **4 字节** 。
*   对整数进行算术运算会返回整数值（例如，`4 / 3` 的结果为 `1`）。

```c
int count = 10;
```

数字可以用不同的进制表示：

*   十进制（基数为10）：如10、25、100等普通数字。
*   二进制（基数为 2）：以 `0b` 为前缀。
*   八进制（基数为 8）：以 `0` 开头。
*   十六进制（基数为 16）：以 `0x` 开头。

```c
int count = 10;      // 十进制
int bin   = 0b1010;  // 二进制（十进制中的 10）
int oct   = 012;     // 八进制（十进制中的 10）
int hex   = 0xA;     // 十六进制（十进制中的 10）
```

#### `long` 和 `long long`

用于整数。在 ESP32 上，`long` 占用 4 字节（与 `int` 相同），而 `long long` 则使用 8 字节来存储超大数值。

```c
long long milliseconds = 10000000000LL;
```

#### `float`

表示单精度浮点数。

*   占用4个字节。
*   用于像 `3.14` 或 `-2.5` 这样的值。

```c
float pi = 3.14f;
```

#### `double`

表示精度更高的十进制数。

*   在 ESP32 上占用 8 字节，用于高精度计算。

```c
double precisePi = 3.1415926535;
```

#### `bool`

表示逻辑值，即真或假。要在 C 语言中使用此类型，必须包含 `<stdbool.h>` 库。内部存储为 `1`（真）或 `0`（假）。

```c
#include <stdbool.h>
bool isLEDOn = true;
```

#### `void`

表示无值。用于不返回任何内容的函数。示例：

```c
void blinkLED() {  
// 此函数不返回任何值  
}
```

#### 固定宽度整数类型

在使用 ESP-IDF 进行嵌入式编程时，强烈建议使用来自 `<stdint.h>` 的固定宽度整数，以确保我们确切知道变量占用的内存大小。

*   `uint8_t`：无符号 1 字节整数（通常用作"字节"）。
*   `int16_t`：有符号 2 字节整数。
*   `uint32_t`：无符号 4 字节整数。

#### 数组

数组是存储在一起的同类型变量的集合。

*   元素通过索引访问，索引从 `0` 开始。
*   数组可以存储整数、字符或其他数据类型。

```c
int numbers[] = {4, 5, 7};           // 整型数组  
char message[] = "hello";            // 字符数组（字符串）  
char name[] = {'A', 'l', 'i', '\0'}; // 手动添加终止符的字符数组
```

### 常量

常量是一种在程序运行期间其值无法改变的变量。我们使用大写字母来命名常量，以便在代码中轻松识别它们。定义常量有两种方式：

*   使用 `const` 关键字：`const float GRAVITY = 9.81;`
*   使用 `#define` 指令：`#define PI 3.14159`

`#define` 是一个预处理器指令，在编译前将名称替换为值，且不进行类型检查；而 `const` 则创建一个类型化的变量，其值不可更改，允许编译器强制执行类型安全。

### 算术运算符

算术运算符用于执行数学运算。它们支持加法、减法、乘法和除法等计算。

*   **加法（+）：** 将两个操作数相加。当与字符类型一起使用时，会将其 ASCII 值相加并返回结果字符。
*   **减法（-）：** 从第一个操作数中减去第二个操作数。与加法类似，当与字符类型一起使用时，会对 ASCII 值进行运算。
*   **乘法（\*）：** 将两个操作数相乘。此运算符不能直接与字符类型一起使用。
*   **除法（/）：**
    *   对于 float 或 double 类型，执行常规除法并返回精确结果。
    *   对于 int 或 long 类型，执行整数除法，会舍弃结果中的小数部分，不会对结果进行四舍五入。
*   **取模（%）：** 返回两个操作数相除后的余数，仅适用于 int 和 long 类型。

### 复合运算符

复合运算符将算术运算与赋值合并为一步，使代码更简洁、清晰且易读。

*   **`+=`（复合加法）：** 将某个值加到变量上，并将结果重新赋值给该变量。`a += 5;` 等同于 `x = x + 5;`  
*   **`-=`（复合减法）：** 从变量中减去某个值。`x -= 2;` 等同于 `x = x - 2;`    
*   **`*=`（复合乘法）：**  将变量乘以一个值。`x *= 4;`  等同于 `x = x * 4;`  
*   **`/=`（复合除法）：**  将变量除以一个值。 `x /= 2;`  等同于 `x = x / 2;`  
*   **`%=`（复合取余）：**  存储除法的余数。 `x %= 3;`  等同于 `x = x % 3;`

#### 递增与递减运算符

这些运算符是特殊的复合运算符，用于将变量增加或减少1。它们在循环和计数器中非常常见。

*   **`++`（递增）：**  将变量增加 1。  `i++；`   等同于 `i = i + 1；`    
*   **`--`（递减）：**  将变量减少 1。  `i--；`  等同于 `i = i - 1；`

这些运算符有两种使用形式：

*   **`++i`（前置递增）**：先递增，再使用值。
*   **`i++`（后置递增）**：先使用值，再递增。

### 比较运算符

比较运算符用于比较两个值，并产生一个布尔结果（在标准 C 中为 `true` 或 `false`，或 `1` / `0`）。

| 运算符 | 含义 |
| --- | --- |
| \== | 等于 |
| != | 不等于 |
| \> | 大于 |
| < | 小于 |
| \>= | 大于或等于 |
| <= | 小于或等于 |

### 布尔运算符

布尔运算符帮助我们组合多个条件，形成更复杂的表达式。逻辑运算的结果也是一个布尔值（真或假）。

*   **||（或）：** 如果由 `||` 连接的至少一个条件为真，则返回 true。
*   **&&（与）：** 仅当由 `&&` 连接的所有条件都为真时，才返回 true。
*   **!（非）：** 反转条件的逻辑状态。

| A | B | A && B | A || B | !A |
| --- | --- | --- | --- | --- |
| true | true | true | true | false |
| true | false | false | true | false |
| false | true | false | true | true |
| false | false | false | false | true |

### 条件语句

#### 使用 `if 的单一条件`

有时我们只需要在条件为真时执行一条指令。如果条件为假，系统会直接跳过该代码块，继续运行程序的其余部分。

```c
if (条件){
指令
}
```

#### 使用 `if-else 的替代路径`

我们也可以提供一条替代路径。如果条件为真，则执行一个代码块；否则，执行 `else` 代码块。

```c
if (条件){
条件成立时执行的指令
}else{
条件不成立时执行的指令
}
```

#### 使用 `else if 处理多个条件`

当存在两个以上的可能结果时，我们可以使用 `else if` 来串联条件。C 语言会按顺序检查这些条件，并执行第一个为真的条件对应的代码块。

```c
if (条件 1){
如果条件1成立则执行的指令
}else if(条件 2){
如果条件2成立，则执行以下指令
}否则{
如果以上条件均不成立，则执行以下指令
}
```

#### 三元运算符

在 C 语言中，我们可以使用三元运算符 `? :`，它是 if-else 语句的简写形式，用于根据条件为变量赋值。该运算符适用于简短且简单的判断场景，尤其是在进行赋值操作时。

```c
status = condition ? "条件为真时的值" : "条件为假时的值";
```

#### `switch``case` 语句

当需要将一个值与多个选项进行比对时，我们可以使用 `switch` 语句。与多个 `else if` 条件相比，它能让代码更简洁。

```c
switch (变量){
case 值 1:
当变量等于值1时执行的指令
break;
case value2:
如果变量等于 value2 则执行的指令
break;
case value3:
如果变量等于 value3 时执行的指令
break;
case value4:
当变量等于 value4 时执行的指令
break;
default:
当变量为其他未提供的值时执行的指令
break;
}
```

### 循环

循环帮助我们多次重复执行某个动作。共有三种类型的循环。

#### `for` 循环

当我们知道需要重复执行某段代码多少次时，就会使用 `for` 循环。这在 Esp32 中非常常见，尤其是在控制 LED、传感器或固定次数重复任务时。

```c
for (初始化; 条件; 递增/递减){
要重复执行的指令
}
```

#### `while` 循环

`while` 循环会在条件为真时重复执行。当我们不知道循环会运行多少次时，它非常有用。

```c
while (条件) {
我们希望重复执行的指令
}
```

#### `do-while` 循环

`do-while` 循环与 `while` 类似，但它至少会执行一次，即使条件为假也是如此。

```c
do{
我们想要重复执行的指令
}while (条件);
```

#### Break 和 Continue：

我们可以通过使用关键字 `break` 和 `continue` 来增强对循环的控制：

*   **`break`**：当满足特定条件时，`break` 语句用于立即终止并退出循环。
*   **`continue`**：当满足特定条件时，`continue` 语句用于跳过当前循环的迭代并继续下一次迭代，从而忽略该次迭代中的指令。

### 注释：

注释是编译器会忽略的代码行。它们用于添加解释性说明，提高代码的可读性。创建注释有两种方式：

*   **单行注释：** 以 `//` 开头，持续到当前行结束。同一行中 `//` 之后的所有文本都会被编译器忽略。
*   **多行注释：** 以 `/*` 开头，以 `*/` 结束。这两个符号之间的任何文本（包括多行内容）都会被编译器忽略。

## 创建我们的第一个项目

现在我们了解了 ESP32-S3 微控制器是什么，并回顾了 C 语言的基本概念，就可以开始为 ESP32-S3 编写一个简单的程序了。

嵌入式系统领域的传统入门项目是 LED 闪烁。这个项目有助于我们验证：

*   开发环境是否正常工作
*   代码能否成功编译
*   该开发板可以控制其 GPIO 引脚

在编写任何代码之前，我们需要设置 ESP‑IDF 所使用的项目文件夹结构。

### 设置项目文件夹

首先，创建一个文件夹，用于存放与项目相关的所有文件。ESP-IDF 项目遵循标准的目录结构，使构建系统能够正确编译和组织我们的代码。

一个最小的 ESP-IDF 项目结构如下所示：

```
my-new-project/
├── CMakeLists.txt（顶层）
└── main/
├── CMakeLists.txt
└── main.c
```

*   `my-new-project` 代表项目的根文件夹，其中包含定义整个项目的配置文件和子文件夹。
*   `CMakeLists.txt` 定义了项目的全局构建设置，我们在其中指定 ESP-IDF 构建系统的位置、项目包含哪些组件以及如何组织构建过程。

```txt
# CMake 的最低版本要求
cmake_minimum_required(VERSION 3.16)

# 包含 ESP-IDF 构建系统
include($ENV{IDF_PATH}/tools/cmake/project.cmake)

# 我们的项目名称
project(my-new-project)
```

*   `main/` 在 ESP-IDF 中，项目被组织成组件，这些组件是处理特定任务的小型构建模块。`main` 文件夹是默认组件，包含我们的主应用程序代码。在小型项目中，所有代码都可以保留在此文件夹内。
*   `main/CMakeLists.txt` 这是组件级别的构建配置文件，它告诉构建系统哪些源文件属于该组件。

```
idf_component_register(SRCS "main.c"
INCLUDE_DIRS ".")
```

我们需要在此文件中设置两个主要内容—— `SRCS` 列出要编译的源文件 - `INCLUDE_DIRS` 指定包含头文件的目录

*   `main/main.c` 此文件包含将在 ESP32-S3 上运行的实际程序代码。

让我们应用这一点，创建 `blink` 文件夹，并在其中添加我们上面讨论过的文件。

### 创建电路

现在我们已经设置好了项目文件夹，让我们来创建电路。我们需要一个 LED（发光二极管）。LED 只允许电流单向流动，并且有两个引脚：

*   **较长的引脚（阳极）：** 连接到正极（信号或电源）。
*   **短脚（阴极）：** 连接至**地线（GND）**。

我们还需要一个 220 欧姆的电阻。该电阻用于限制流经 LED 的电流，防止其烧毁。
为了控制 LED 并使其闪烁，我们将使用 ESP32 上的数字 GPIO 引脚。在本示例中，我们将使用 GPIO 2。

现在开始构建我们的电路：

1.  将 LED 的长脚（阳极）通过 220 欧姆电阻连接到 GPIO 2。
2.  将 LED 的短脚（阴极）连接到 ESP32 上的 GND 引脚。

### 创建程序

现在我们来为 ESP32 开发板编写程序。我们打开 `main/main.c` 文件，首先导入所需的库。

```c
#include <stdio.h>
#include "freertos/FreeRTOS.h"
#include "freertos/task.h"
#include "driver/gpio.h"
```

**`#include <stdio.h>`** 是 C 语言中的标准输入/输出库。它允许我们使用诸如 `printf()` 之类的函数将消息打印到串行监视器。

**`#include "freertos/FreeRTOS.h"`** 该头文件包含了 ESP-IDF 框架使用的 FreeRTOS 核心定义。FreeRTOS 是运行在 ESP32 上的实时操作系统，负责管理任务、时序和系统资源。

**`#include "freertos/task.h"`** 该库提供了在 FreeRTOS 中创建和管理任务的函数。它还包含一些有用的函数，例如 `vTaskDelay()`，允许我们将任务暂停特定时间。

**`#include "driver/gpio.h"`** 该头文件使我们能够访问 ESP32 的 GPIO（通用输入/输出）驱动程序。它允许我们将引脚配置为输入或输出并进行控制，这正是我们通过 GPIO 2 来打开和关闭 LED 的方式。

现在我们创建 `app_main` 函数，首先设置引脚的模式，然后

```c
void app_main(void){
gpio_reset_pin(2);
gpio_set_direction(2, GPIO_MODE_OUTPUT);
```

之后，我们创建一个无限 `while` 循环。在这个循环内部，我们通过向 GPIO 2 发送信号来控制 LED。
为了打开和关闭 LED，我们使用函数： `gpio_set_level(PIN, signal);`，该函数用于设置指定 GPIO 引脚的电气电平。

*   **`PIN`** 代表我们要控制的 GPIO 编号（在本例中为 **GPIO 2**）。
*   **`signal`** 决定输出电平：
    *   `1` → 将 LED 灯**打开**
    *   `0` → 将 LED 灯**关闭**

为了产生闪烁效果，我们在打开和关闭 LED 灯之间添加了一个延迟，使用：`vTaskDelay(1000 / portTICK_PERIOD_MS);` 让任务暂停特定时长。

*   **`1000`** 表示延迟时间，单位为**毫秒（1 秒）**。
*   **`portTICK_PERIOD_MS`** 将毫秒转换为 **FreeRTOS 系统时钟节拍** ，这是操作系统使用的时间单位。

```c
while (1) {
gpio_set_level(2, 1);
vTaskDelay(1000 / portTICK_PERIOD_MS);
gpio_set_level(2, 0);
vTaskDelay(1000 / portTICK_PERIOD_MS);
    }
}
```

现在让我们执行程序并将其烧录到微控制器中。

#### 连接并设置目标设备

首先通过 USB 连接微控制器。接着选择目标微控制器（ESP32-S3），以便编译器知道使用哪种芯片架构：

```shell
idf.py set-target esp32s3
```

### 识别端口

接下来，我们检查微控制器连接到了哪个端口。我们可以通过扫描连接的硬件来找到它：

```shell
esptool.py chip_id
```

在 Windows 上，这看起来像 `COM3`；在 Linux/macOS 上，则像 `/dev/ttyUSB0` 或 `/dev/cu.usbmodem...`。

#### 构建项目

之后，我们将代码编译为机器可读的格式。这一步会检查 C 代码中的错误：

```shell
idf.py build
```

#### Flash 与 Monitor

最后，我们烧录并监控程序。

```
idf.py -p PORT flash monitor
```

将 `PORT` 替换为你在步骤 2 中找到的端口 ID。
